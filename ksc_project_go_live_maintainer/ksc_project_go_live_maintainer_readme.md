=====================
Detailed Information
=====================
Follow steps to avail functionality cap requirements:
Note===> .
(1) The KSC Project Go Live Maintainer module focuses on facilitating Go Live Change Requests for project modifications.
(2) in the Go Live Change Request Steps,on company configuration, when selecting the option for one step,the Step 1 Validator will be displayed. Alternatively, when selecting two steps, both Step 1 Validator and Step 2 Validator will be shown.
(3) In the project interface, click on "Go Live Change Request" to reveal fields such as Project, Reason, Written Proof, Current Go Live Date,
and New Go Live Date. Upon clicking the "New" button, a form view opens, incorporating all necessary fields, with the Current Go Live Date field set as read-only.
(4) Initially, only the "Validate" button is displayed. However, when selecting the first step for a project, only the "Validate" button is available.
In contrast, upon selecting the second step, the "Approve" button becomes visible.
(5) when configuring a two-step project, the Step 1 Validator approves the request before forwarding it to the Step 2 Validator.
